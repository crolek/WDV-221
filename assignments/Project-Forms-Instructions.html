Project - Forms Instructions

The goal of this project is to learn how to validate a form with plain JS.

I created an HTML form for you to use to complete this project.

The form does not have to Submit to any location. It just needs to console.log() the resulting data.

The result object that you console.log() should be structured like: 

    {
        "firstName": "Stewie",
        "email": "lilsquiggle@gmail.com",
        "message": "Something goes here",
        "gender": "male"
    }

The exact keys of "firstName", "email", "message", and "gender" have to be used for the object that you console.log().

There are 4 different input types on the page and each needs to be validated before the form can "submit" (aka, console.log() the results). You can validate the individual fields on a blur event, or all at once when the user clicks the "submit" button.

If the field is not valid (like the First Name field) you should display an appropriate message in the error-box. The message could be "First Name field is incorrect, it can only have letters.". You must have an unique error message per field.
The error messages should go in the div with the class of "error-box".
Error messages should disappear when the field becomes a valid field.

Hint:
 You will have to use the event.preventDefault() function inside your event handler to complete this project. Reference the preventDefault api.


Field Validation Rules

Name:

  1. It cannot be empty
  2. It cannot have any numbers in it
  3. It can have a dash in it

Email:

  1. It cannot be empty
  2. It must have the @ symbol in it
  3. It must have at least 1 period in it on the right side of the @ symbol

Message:

  1. It cannot be empty
  2. It must have more than 3 characters in it

Gender:

  1. One of the radios must be selected
